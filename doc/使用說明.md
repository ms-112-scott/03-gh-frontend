# 03-gh-frontend 使用說明

## 專案定位

`03-gh-frontend` 是 Grasshopper / Rhino 端與 Taichi / NCA 執行流程之間的前端橋接專案，負責：

- 接收 Grasshopper 幾何與控制參數
- 透過 WebSocket 與本地執行中的 frontend runtime 溝通
- 使用 Taichi 產生 mask / SDF 等中介資料
- 載入來自 `02-nca-cfd` 的模型 checkpoint
- 將 NCA 輸出轉為可視化結果與統計資訊

## 主要檔案位置

- [main.py](/C:/Users/User/Desktop/NCA_workspace/03-gh-frontend/src/app/main.py)
  主要執行入口，負責啟動 WebSocket server、GUI 與主流程。
- [grasshopper_client.py](/C:/Users/User/Desktop/NCA_workspace/03-gh-frontend/src/gh/grasshopper_client.py)
  Grasshopper 端腳本，負責傳送 payload 與解析回應。

  幾何轉 mask。

  幾何轉 SDF。

## 執行前準備

### Python 套件

建議使用 repo 內的 `.venv` 虛擬環境，並依 [requirements.txt](/C:/Users/User/Desktop/NCA_workspace/03-gh-frontend/requirements.txt) 安裝。

主要會用到：

- `numpy`
- `opencv-python`
- `matplotlib`
- `PyYAML`
- `taichi`
- `torch`
- `websockets`
- `websocket-client`
- `pathspec`

### `02-nca-cfd` 依賴

本專案會讀取 `02-nca-cfd` 內的模型 checkpoint 與模型原始碼，因此需保留以下關係：

- `03-gh-frontend` 與 `02-nca-cfd` 位於同一個 workspace 層級
- `02-nca-cfd` 僅供讀取，不在此 repo 修改

### 設定檔

主要設定位於 [config.yaml](/C:/Users/User/Desktop/NCA_workspace/03-gh-frontend/config.yaml)。

重要欄位如下：

- `ws_host`
- `ws_port`
- `data_res`
- `gui_res`
- `max_edges`
- `nca_checkpoint`
- `nca_cfd_src`
- `nca_steps_per_frame`
- `lbm_to_ms`
- `default_viz_mode`

## 啟動方式

### 建立與啟用虛擬環境

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

### 使用 npm script 啟動

[package.json](/C:/Users/User/Desktop/NCA_workspace/03-gh-frontend/package.json) 目前提供：

```bash
npm start
```

實際會執行：

```bash
.\.venv\Scripts\python.exe -m src.app.main
```

### 直接使用 Python 啟動

也可以直接在 repo 根目錄執行：

```bash
.\.venv\Scripts\python.exe -m src.app.main
```

啟動後會：

- 載入 `config.yaml`
- 初始化 Taichi
- 嘗試載入 `nca_checkpoint`
- 啟動 WebSocket server
- 開啟 Taichi GUI 畫面

## Grasshopper 端使用方式

Grasshopper 端主要腳本位於 [grasshopper_client.py](/C:/Users/User/Desktop/NCA_workspace/03-gh-frontend/src/gh/grasshopper_client.py)。

常用輸入包含：

- `run`
- `polyline_tree`
- `boundary`
- `mode`
- `viz_mode`
- `img_save_path`

基本流程：

1. 先啟動 `03-gh-frontend` runtime。
2. 在 Grasshopper 的 GHPython component 中使用對應腳本。
3. 將幾何與控制參數送出。
4. 腳本透過 WebSocket 與本地 server 溝通。
5. 取得狀態、統計資料、圖例或影像輸出。

## 視覺化模式

目前主要支援以下 `viz_mode`：

- `velocity`
- `vorticity`
- `pressure`

- `stress`
